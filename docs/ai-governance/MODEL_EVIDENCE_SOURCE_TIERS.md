# 模型评估证据来源分级标准

本文件用于定义模型评估时可采纳的证据来源等级，避免将博客、自媒体、排行榜页面直接当作正式准入依据。

本标准适用于：

1. 候选模型选型
2. 候选模型测试结果解释
3. 主线编码准入判断
4. 模型长期观测与复盘

## 1. 目标

模型评估时常见的问题不是“没有资料”，而是“资料来源等级混乱”。

例如：

1. 厂商官网说明
2. benchmark 官方 leaderboard
3. 聚合排行榜
4. 知乎、掘金、博客园、自媒体

这些来源的可信度、可复核性和适用范围不同，不能混用。

## 2. A 级来源：正式判定依据

以下来源可进入正式证据链，可用于支持模型档位、准入结论和治理结论：

### 2.1 官方 benchmark / 原始论文 / 官方 leaderboard

示例：

1. `SWE-bench Verified`
2. `LiveCodeBench`
3. `Terminal-Bench`
4. `BFCL`
5. benchmark 原始论文和官方 leaderboard

### 2.2 官方云平台 / 官方产品文档

示例：

1. 阿里云百炼模型评测与模型监控文档
2. 华为云 ModelArts 精度评测文档
3. 智谱官方模型评测文档
4. DeepSeek 官方 API / coding agent 文档

### 2.3 官方研究机构 / 行业研究报告

示例：

1. 信通院《大模型基准测试体系研究报告》

### 2.4 项目内自建证据

示例：

1. 候选模型测试题库
2. 越界判定记录
3. 项目门禁日志
4. 候选补丁复审结论
5. 模型准入档位记录

### 2.5 使用规则

A 级来源可用于：

1. 模型档位判定
2. 是否继续调教
3. 是否进入受控编码
4. 是否进入主线编码

## 3. B 级来源：候选导航依据

以下来源可用于筛选候选模型、查看市场热度或交叉参考，但不能单独作为正式准入依据：

### 3.1 聚合排行榜 / 聚合评测导航

示例：

1. `OpenRouter rankings`
2. `DataLearner leaderboards`
3. 其他 benchmark 聚合导航站点

### 3.2 价值

可用于：

1. 查看当前热门模型
2. 查看不同榜单的聚合结果
3. 决定“是否值得继续测”

不可用于：

1. 直接判定主线可用
2. 直接判定模型已通过项目准入
3. 覆盖项目门禁结果

### 3.3 使用规则

B 级来源只能作为：

```text
候选导航 / 热度参考 / 继续调教优先级参考
```

不能单独写成：

```text
模型已通过准入
模型可直接进入主线
```

## 4. C 级来源：辅助理解资料

以下来源可用于理解概念、学习方法，但不能进入正式判定证据链：

示例：

1. 知乎文章
2. 掘金文章
3. 博客园文章
4. 自媒体解读
5. 二手转述材料

### 4.1 价值

可用于：

1. 理解评估概念
2. 学习指标含义
3. 辅助整理方法论

不可用于：

1. 直接支持模型准入结论
2. 直接支持模型档位提升
3. 直接反驳项目内门禁结果

## 5. 来源冲突时的优先级

若多个来源结论冲突，按以下顺序取信：

1. 项目内红线门禁与测试记录
2. A 级正式来源
3. B 级聚合来源
4. C 级辅助理解资料

即：

```text
项目内真实测试 > 官方正式来源 > 聚合排行榜 > 自媒体/博客
```

## 6. 使用示例

### DoubaoCode

```text
即使有外部热度或厂商宣传，只要项目内门禁频繁失败，仍判为不合格
```

### DeepSeek v4

```text
即使聚合排行榜或厂商说明较强，也只能说明“值得继续测”
真正档位仍由项目题库、门禁和补丁复审决定
```

## 7. 统一结论模板

在引用任何外部来源时，建议按以下格式说明：

```text
来源名称：
来源级别：A / B / C
用途：
可支持的结论：
不可支持的结论：
```

示例：

```text
来源名称：OpenRouter rankings
来源级别：B
用途：查看市场采用度和候选模型热度
可支持的结论：值得继续测试
不可支持的结论：已通过主线编码准入
```
